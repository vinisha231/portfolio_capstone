Annotated Bibliography

Resources are sorted by theme (alphabetically).

API and Backend Development

I needed a backend that could define clear routes, validate requests and responses, and plug in database and auth cleanly. I chose FastAPI for its automatic OpenAPI docs and type-driven design; Pydantic made request/response shapes explicit and caught errors early. Uvicorn was the obvious choice to run the ASGI app. Together they worked well—the router layout stayed readable and adding new endpoints stayed straightforward. The list of sources I relied on for this cluster:

  • Pydantic. (2024). Pydantic documentation. https://docs.pydantic.dev
  • Tiangolo, S. (2024). FastAPI. https://fastapi.tiangolo.com
  • Uvicorn. (n.d.). Uvicorn. https://www.uvicorn.org

AWS and Cloud Services

Bedrock was the main integration point for question generation and evaluation, so I needed to get the runtime client, region, and model IDs right and to handle both Claude (Messages API) and Llama (prompt-based) formats. The Bedrock and Boto3 docs were essential for that; they were sometimes dense but accurate. I also used AWS docs for Elastic Beanstalk deployment and for optional services like Transcribe and Polly. The sources that supported this work:

  • Amazon Web Services. (2024). Amazon Bedrock developer guide. https://docs.aws.amazon.com/bedrock/
  • Amazon Web Services. (2024). AWS SDK for Python (Boto3). https://boto3.amazonaws.com/v1/documentation/api/latest/index.html
  • Amazon Web Services. (2024). Elastic Beanstalk developer guide. https://docs.aws.amazon.com/elasticbeanstalk/

Databases and Data Modeling

I picked PostgreSQL for familiarity and for running on RDS or similar in production. SQLAlchemy gave me an ORM and session lifecycle that fit FastAPI’s dependency-injection style; Alembic made migrations manageable so schema changes were repeatable. The docs were strong on schema design (foreign keys, nullable fields, timestamps) and on loading DATABASE_URL from env in local and cloud setups. The references I used:

  • PostgreSQL Global Development Group. (2024). PostgreSQL 16 documentation. https://www.postgresql.org/docs/16/
  • The SQLAlchemy Authors. (2024). SQLAlchemy documentation. https://docs.sqlalchemy.org

Frontend and UX

I wanted a frontend that could handle the setup → question → answer → feedback flow and talk reliably to the FastAPI backend. React’s component model and state handling fit that; I used the official React docs for hooks and data flow. MDN was my go-to for HTTP and fetch behavior when wiring the client to the API. Both worked well for implementing the interview and dashboard and for thinking about UX and accessibility. Sources:

  • Meta. (2024). React. https://react.dev
  • Mozilla Developer Network. (2024). HTTP. https://developer.mozilla.org/en-US/docs/Web/HTTP

LLMs and Prompting

I needed to design prompts that produced good interview questions and evaluation feedback and that returned parseable JSON. Claude’s docs were especially useful for system vs. user prompts and for constraining output; the Bedrock InvokeModel reference was necessary for request shape and model IDs. I used them to build type-specific rubrics (technical, behavioral, design) and to refine prompts until the model returned consistent scores and suggestions. The sources that helped:

  • Anthropic. (2024). Claude documentation. https://docs.anthropic.com
  • Amazon Web Services. (2024). InvokeModel – Bedrock runtime. https://docs.aws.amazon.com/bedrock/latest/APIReference/API_runtime_InvokeModel.html
