Annotated Bibliography

Resources are sorted by theme (alphabetically). Each theme has a short paragraph explaining that cluster and how it informed the capstone work, followed by APA-formatted citations in alphabetical order.

API and Backend Development

This cluster supported building the FastAPI backend: defining routes and routers, request/response models with Pydantic, dependency injection for database sessions and auth, CORS for the React frontend, and running the app with Uvicorn. These resources were used to implement the interview, sessions, dashboard, and auth endpoints and to structure the code for clarity and testability.

  • Pydantic. (2024). Pydantic documentation. https://docs.pydantic.dev
  • Tiangolo, S. (2024). FastAPI. https://fastapi.tiangolo.com
  • Uvicorn. (n.d.). Uvicorn. https://www.uvicorn.org

AWS and Cloud Services

These resources were used to integrate AWS Bedrock for LLM-based question generation and answer evaluation, to configure the Bedrock runtime client (region, model ID), and to understand the Claude and Llama invocation formats. Additional AWS services in the project (e.g. Transcribe, Polly) and deployment on Elastic Beanstalk also drew on AWS documentation.

  • Amazon Web Services. (2024). Amazon Bedrock developer guide. https://docs.aws.amazon.com/bedrock/
  • Amazon Web Services. (2024). AWS SDK for Python (Boto3). https://boto3.amazonaws.com/v1/documentation/api/latest/index.html
  • Amazon Web Services. (2024). Elastic Beanstalk developer guide. https://docs.aws.amazon.com/elasticbeanstalk/

Databases and Data Modeling

These resources supported designing and implementing the persistence layer: PostgreSQL as the database, SQLAlchemy for ORM and session management, Alembic for migrations, and the User and InterviewSession models. They informed schema design (e.g. foreign keys, nullable fields, timestamps) and how to load configuration (e.g. DATABASE_URL) across local and cloud environments.

  • PostgreSQL Global Development Group. (2024). PostgreSQL 16 documentation. https://www.postgresql.org/docs/16/
  • The SQLAlchemy Authors. (2024). SQLAlchemy documentation. https://docs.sqlalchemy.org

Frontend and UX

These resources supported building the React frontend: components, state management, and calling the FastAPI backend. They informed the interview flow (setup, question/answer, feedback), the dashboard (stats and history), and general UX and accessibility practices for a portfolio and for the simulator itself.

  • Meta. (2024). React. https://react.dev
  • Mozilla Developer Network. (2024). HTTP. https://developer.mozilla.org/en-US/docs/Web/HTTP

LLMs and Prompting

These resources informed how the capstone uses LLMs via AWS Bedrock: designing system and user prompts for question generation, structuring evaluation prompts with rubrics (technical, behavioral, design), and requesting structured JSON output for scores and feedback. They also supported understanding model behavior and prompt-engineering practices for interview-style tasks.

  • Anthropic. (2024). Claude documentation. https://docs.anthropic.com
  • Amazon Web Services. (2024). InvokeModel – Bedrock runtime. https://docs.aws.amazon.com/bedrock/latest/APIReference/API_runtime_InvokeModel.html
